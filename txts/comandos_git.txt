ðŸ”¹ Flujo de Trabajo Recomendado
1. Trabajar normalmente en `main` para cambios pequeÃ±os o seguros.
2. Crear una rama (`git branch <rama> && git switch <rama>`) para cambios grandes o inestables.
3. Hacer commits en la nueva rama.
4. Una vez estable, hacer merge a `main` y continuar desde ahÃ­.
5. No eliminar la rama inmediatamente si usas varias mÃ¡quinas.
6. Cuando regreses a otra mÃ¡quina, hacer `git fetch` o `git pull` para mantenerla sincronizada.
7. MantÃ©n `.gitignore` actualizado para excluir archivos como `.pyc` y carpetas `__pycache__`.
